正则表达式

用来操作字符串（擅长匹配模糊范围）

 处理起来更高效

 维护不方便，可读性不高

两种写法

 标准：new RegExp(规则，修饰符)

 简写：/规则/

  性能比标准的好

\ 转义符

 \d 代表一个数字； \D 代表一个非数字；

 \w 代表一个字符（数字，字母，下划线）； \W 代表非字符

 \s 代表空格； \S代表非空格

 \b 边界符

  不适用中文

  . 代表任意字符除了\n

修饰符

  i 忽略大小写

  g 查找全局

正则.test( str );

 检测正则匹配的内容

 如果匹配到内容 返回 true 反之 false

 例子：console.log(/\d/.test(str))

str.match(正则);

 如果匹配成功将匹配到的字符放到一个数组中，匹配不成功返回null;

 当匹配到一个字符时，会显示多个属性

  index: 匹配字符的位置

  input: 所有内容

  而length为1

str.search(字符串||正则);

 找到匹配的字符位置（可以指定也可以正则匹配）

 如果找不到返回值 = -1；

str.replace( 要替换谁 || 可以用正则 , 替换成什么 || 函数 );

 返回值为替换好的字符串

  | 或者

 如果使用函数，那么一定要有return,不然为undefind

 一般情况下 函数内的第一个参数是当前匹配到的字符。 函数内的第二个参数是当前匹配到的字符的索引值。 函数内的第三个参数是当前总的字符串。 函数内的第四个以后参数都为undefined。

子项

 如果正则中有子项，从函数的第一个参数之后就是匹配到的子项值，当参数的个数等于子项个数+1的时候，之后都为正常模式下的参数了(比如：索引，整个符，undefined..)

 \1 子项重定向 （重复\number前面一个字符）

 当有子项的时候，如果在子项的后面加个\+数字，这个\+数字的意思就是重复一次前面子项中的字符，这个\+数字就叫子项重定向子项重定向可以在后面加量词

( ) : 提权

[ ] : 在中括号里任意选择一个字符。

 [^a] 排除a

 在中括号中 + ^ = 排除中括号中的内容。

 如果把^放到正则开始的位置，代表开头 结束$ /^ab$/

 [\u4e00-\u9fa5] ; 中文的范围

{ } 量词的范围

  {0,1} ？ 最小可以没有最大1个：

  {0，} * 最小没有，最大无限

  {1，} + 最小1个最大无限

  {n} 固定的次数
